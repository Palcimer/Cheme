C# 스레드:

Thread t = new Tread(함수 이름);
t.Start(); : 스레드 시작
t.isBackground=true : 스레드를 백그라운드에서 실행(호출한 함수가 종료될 때 같이 종료), 기본값: flase
t.Join(); : 백그라운드여도 스레드가 끝날 때까지 기다림, task 객체라면 t.Wait로 씀
ThreadPool.QueueUserWorkItem(함수 이름); : 스레드를 새 객체로 안 만들고 간단하게 쓰는 건가? 모르겠음... 백그라운드 실행
백그라운드 실행이면 매번 스레드가 어디까지 실행되고 끝나는가가 달라짐
스레드풀은 가급적 간단한 함수에 사용
Task는 더 오래걸리는 작업에 사용
